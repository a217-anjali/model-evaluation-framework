# 03 — Ministerial Sign-Off

## Sources

| Source | Reference |
|--------|-----------|
| ETA Framework | Point 3: Ministerial approval required for significant automated decisions |

## Purpose

The ETA Framework requires ministerial approval for all significant automated decisions. This section defines what constitutes "significant" and the approval process.

## When Ministerial Sign-Off is Required

- [ ] The LLM makes or materially influences decisions with legal effects on citizens
- [ ] The LLM processes personal data at significant scale
- [ ] The LLM is deployed in a politically sensitive area
- [ ] The deployment represents a novel use of AI in the department
- [ ] The spend exceeds departmental thresholds for technology investment

## Briefing Requirements

The ministerial briefing should cover:

1. **What**: Description of the LLM and its role in plain language
2. **Why**: The user need and public benefit
3. **Risks**: Key risks identified through the evaluation process
4. **Mitigations**: How risks are being managed
5. **Testing**: Summary of red-teaming, bias testing, and security testing results
6. **Governance**: Who is accountable and how the system is overseen
7. **Transparency**: What is being published (ATRS record, performance data)
8. **Contestability**: How citizens can challenge decisions
9. **Legal**: Legal compliance confirmation
10. **Recommendation**: Go / No-Go / Conditional with rationale

## Output

Signed ministerial submission with decision recorded.
