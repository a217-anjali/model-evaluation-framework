## Purpose

Contracts with LLM providers must include specific provisions beyond standard IT procurement. The DEF states: "the public body remains responsible for all data and algorithm-assisted decisions" — responsibility cannot be outsourced.

## Required Contract Provisions

### Transparency

- [ ] Supplier must provide information needed to complete ATRS records
- [ ] ATRS information categories referenced in contract
- [ ] Supplier must disclose significant model updates or changes
- [ ] Supplier must provide model documentation (model card or equivalent)

### Data and privacy

- [ ] Data processing agreement in place (UK GDPR compliant)
- [ ] Data residency requirements specified (where is data processed?)
- [ ] Prohibition on using government data to train supplier's models (unless agreed)
- [ ] Data deletion provisions on contract termination
- [ ] Supplier's sub-processors disclosed

### Ethical standards (per DEF)

- [ ] Supplier must meet equivalent ethical standards for transparency
- [ ] Accountability through contractual liability for algorithmic outputs
- [ ] Fairness testing and bias mitigation obligations
- [ ] Documentation of system limitations
- [ ] Right to audit the supplier's AI practices

### Knowledge transfer (per PROC Consideration 10)

- [ ] Knowledge transfer provisions included
- [ ] Staff training provided by supplier
- [ ] Documentation of integrations and configurations
- [ ] Exit strategy defined (how to move to a different provider)

### Ongoing management

- [ ] SLAs for model performance, uptime, and response time
- [ ] Incident response and notification requirements
- [ ] Regular performance reporting from supplier
- [ ] Right to conduct independent testing / red-teaming
- [ ] Model version management and change notification

### Liability

- [ ] Liability allocation for AI-caused harm
- [ ] Indemnification provisions
- [ ] Insurance requirements

## Key Principle

> "The public body remains responsible for all data and algorithm-assisted decisions" — Data & AI Ethics Framework

This means the contract must give the organisation sufficient control and visibility to discharge this responsibility.

## Output

1. AI-specific contract schedule / addendum
