## Purpose

The ETA Framework establishes a "presumption of publication" — all algorithmic tools used in government decisions should be explained in plain language. Citizens should not need technical knowledge to understand how AI affects them.

## Requirements

### Public-facing explanation

- [ ] Plain-language description of what the LLM does (no jargon)
- [ ] Explanation of how the LLM contributes to decisions
- [ ] Clear statement of what the LLM cannot do
- [ ] Information on how to get a human review
- [ ] Explanation published and easily accessible

### Writing standards

- [ ] Written at a reading level accessible to the general public
- [ ] Tested with actual users for comprehension
- [ ] Available in accessible formats (screen reader compatible, large print if needed)
- [ ] Welsh language version if applicable
- [ ] Consistent with GOV.UK content design standards

### What must be communicated

1. **What**: "We use an AI tool to help [describe function]"
2. **How**: "It works by [simple description of process]"
3. **Limits**: "It cannot [describe key limitations]"
4. **Human role**: "A human always [describe oversight]"
5. **Your rights**: "If you disagree, you can [describe challenge process]"
6. **Contact**: "If you have questions, contact [named officer or team]"

### ETA Framework "presumption of publication"

The default position is that information about algorithmic tools is published. Exemptions must be justified (e.g., national security). The burden is on the organisation to justify non-publication, not on citizens to request information.

## Output

1. Published plain-language explanation
2. User-tested for comprehension
3. Integrated into the service (not a separate document users must find)
