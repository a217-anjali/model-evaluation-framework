## Purpose

Plan for the orderly decommissioning of the LLM when it is no longer needed, no longer performs adequately, or is being replaced.

## Triggers for Decommissioning

- Model no longer meets performance thresholds
- Better alternative available
- Use case no longer exists
- Unacceptable bias or safety issues cannot be remediated
- Supplier discontinues the model
- Regulatory changes make the deployment non-compliant
- Cost-benefit no longer justified

## Decommissioning Requirements

### Data handling

- [ ] Personal data processed by the LLM is handled per retention policy
- [ ] Data deletion from supplier systems confirmed (per contract)
- [ ] Audit logs retained per retention requirements
- [ ] DPIA updated to reflect decommissioning

### Service continuity

- [ ] Users are informed of the change
- [ ] Alternative service path is in place before decommissioning
- [ ] Transition period planned if replacing with another system
- [ ] No citizen is left without service during transition

### Knowledge preservation

- [ ] Lessons learned documented
- [ ] Performance data archived
- [ ] Decision records preserved per retention policy
- [ ] Knowledge transfer from supplier completed (per contract)

### Governance

- [ ] Decommissioning approved by SRO
- [ ] ATRS record updated (marked as decommissioned)
- [ ] Governance structure stood down
- [ ] Final quarterly review conducted

## Output

Decommissioning plan and completion report.
