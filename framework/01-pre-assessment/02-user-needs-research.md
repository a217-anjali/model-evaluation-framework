## Purpose

The Service Standard (Point 1) requires that all services start with understanding user needs. For LLM-based services, this includes understanding user attitudes toward AI, their ability to assess AI-generated content, and their accessibility requirements.

## Requirements

### User research activities

- [ ] Conducted discovery research with actual end users
- [ ] Identified user needs that the LLM is intended to address
- [ ] Tested whether users can distinguish AI-generated from human-generated content
- [ ] Assessed user trust levels and attitudes toward AI in this context
- [ ] Identified accessibility needs specific to AI interactions
- [ ] Researched what happens when the LLM gets it wrong — how do users recover?
- [ ] Tested with users from diverse demographics and abilities

### Key questions to answer

1. What are users trying to do, and how does the LLM help?
2. Do users understand they are interacting with AI?
3. Can users identify when the LLM output is incorrect?
4. What do users do when the AI fails them?
5. Are there users who cannot or choose not to use the AI — what is their alternative path?

### Service Standard Point 2: Whole problem

- [ ] The LLM fits within the broader user journey, not as an isolated feature
- [ ] Non-AI channels (phone, in-person, letter) remain consistent with AI outputs
- [ ] Users are not left stranded when the LLM cannot handle their case

## Output

A user research report covering:
1. User needs identified
2. User attitudes toward AI in this service
3. Accessibility and inclusion findings
4. Failure mode research (what happens when AI is wrong)
5. Recommendations for human fallback paths
