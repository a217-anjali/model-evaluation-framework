# Glossary

Key terms used throughout this framework.

---

| Term | Definition |
|------|-----------|
| **AISI** | AI Safety Institute — UK state-backed body for evaluating advanced AI systems |
| **ALB** | Arm's-Length Body — public body operating at arm's length from government |
| **ATRS** | Algorithmic Transparency Recording Standard — mandatory disclosure standard for government |
| **CDDO** | Central Digital and Data Office — part of Cabinet Office |
| **CDEI** | Centre for Data Ethics and Innovation — now absorbed into DSIT as RTA |
| **Counterfactual testing** | Testing LLM with identical inputs except for demographic indicators to detect bias |
| **DEF** | Data and AI Ethics Framework — 7-principle ethical framework |
| **Demographic parity** | Fairness metric: equal positive outcome rates across demographic groups |
| **DPIA** | Data Protection Impact Assessment — required under UK GDPR for high-risk processing |
| **DSIT** | Department for Science, Innovation and Technology |
| **EIA** | Equality Impact Assessment — required under Equality Act 2010 |
| **Equalized odds** | Fairness metric: equal true positive and false positive rates across groups |
| **ETA** | Ethics, Transparency and Accountability Framework — 7-point framework for automated decisions |
| **FAST** | Fairness, Accountability, Sustainability, Transparency — Turing Institute principles |
| **GDS** | Government Digital Service |
| **Hallucination** | LLM generating plausible but factually incorrect information |
| **Human-in-the-loop** | Human reviews every AI output before action is taken |
| **Human-on-the-loop** | Human monitors AI outputs and intervenes when needed |
| **ICO** | Information Commissioner's Office — UK data protection regulator |
| **Jailbreak** | Technique to bypass an LLM's safety guardrails |
| **LLM** | Large Language Model — AI system trained on text data to generate natural language |
| **Model card** | Documentation describing a model's capabilities, limitations, and intended use |
| **Model drift** | Gradual degradation in model performance over time |
| **PBG** | Process-Based Governance — Turing Institute governance framework |
| **Prompt injection** | Malicious prompt designed to override an LLM's system instructions |
| **PSED** | Public Sector Equality Duty — Section 149 of Equality Act 2010 |
| **RAG** | Retrieval-Augmented Generation — combining LLM with external data retrieval |
| **Red-teaming** | Adversarial testing by domain experts to find vulnerabilities |
| **RTA** | Responsible Technology Adoption Unit — successor to CDEI within DSIT |
| **SAFE-D** | Sustainability, Accountability, Fairness, Explainability, Data Stewardship — Turing principles |
| **SRO** | Senior Responsible Owner — individual accountable for the AI deployment |
| **SS** | Service Standard — 14-point standard for government digital services |
| **SUM** | Turing Institute values: respect for dignity, connection, wellbeing, public interest |
| **TCoP** | Technology Code of Practice — 13 criteria for government technology |
| **WCAG** | Web Content Accessibility Guidelines — international accessibility standard |
| **WP** | White Paper — refers to the AI Regulation Pro-Innovation Approach white paper |
